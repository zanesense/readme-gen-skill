# OSS README Skill

A simple writing skill for generating clean, professional `README.md` files for open-source projects.

This repo contains a reusable skill that helps AI coding assistants write READMEs with better structure, clearer usage instructions, and a more developer-friendly tone.

## What It Does

The skill guides an assistant to create READMEs that are:

* Clear from the first few lines
* Easy for developers to scan
* Practical, not overly promotional
* Organized with useful sections
* Focused on real project details
* Suitable for open-source repositories

It is especially useful for tools, CLI packages, dashboards, developer utilities, and security-related projects.

## What’s Included

```text
.
└── SKILL.md
```

`SKILL.md` contains the full instructions for writing polished open-source README files, including section ordering, tone rules, examples, and quality checks.

## How to Use

Add this skill to your AI assistant or local coding workflow that supports custom skills.

Then ask it something like:

```text
Write a README.md for this repository.
```

Or:

```text
Generate a simple README for my CLI tool.
```

You can also provide extra context such as:

```text
The project is a defensive GitHub repo scanner.
It has a web dashboard, CLI, JSON export, and npm package.
Keep the README simple.
```

## Best For

This skill works well for:

* Open-source tools
* npm packages
* CLI utilities
* Web apps
* Security tools
* Developer dashboards
* Small to medium GitHub projects

## Customization

You can edit `SKILL.md` to change:

* README section order
* Writing tone
* Required sections
* Badge style
* Security warning format
* Contribution guidelines
* Markdown conventions

Keep the instructions lightweight if you prefer shorter READMEs.

## Goal

The goal of this skill is simple:

> Help developers create READMEs that explain the project clearly, show how to use it quickly, and make the repo look trustworthy.
